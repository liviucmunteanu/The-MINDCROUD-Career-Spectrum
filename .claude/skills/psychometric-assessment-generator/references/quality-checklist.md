# Quality Checklist: 9+/10 Assessment Standards

Run every item and every deliverable through this checklist before delivery.
This checklist operationalizes the difference between a mediocre assessment and a
professional-grade instrument.

## Table of Contents
1. Item-Level Quality (Per Item)
2. Scale-Level Quality (Per Subscale)
3. Instrument-Level Quality (Whole Assessment)
4. Deliverable Quality
5. Ethical Compliance
6. Final Sign-Off Criteria

## 1. Item-Level Quality

Apply to EVERY item. An item must pass ALL applicable checks:

### Clarity
- [ ] Contains exactly ONE idea (no double-barreled items)
- [ ] Written at ≤ 8th grade reading level (short words, short sentences)
- [ ] No jargon, technical terms, or acronyms (unless target population is technical)
- [ ] No idioms, metaphors, or culturally specific references
- [ ] No double negatives
- [ ] No ambiguous pronouns or referents
- [ ] Grammatically correct and complete

### Content
- [ ] Uses behavioral/observable language (not evaluative abstractions)
- [ ] Written in present tense, first person (for self-report)
- [ ] Does not presuppose a particular experience or situation
- [ ] Does not contain loaded or leading language
- [ ] Each item is independently interpretable (doesn't reference other items)
- [ ] Item is relevant to the target construct (face validity)

### Response Format
- [ ] Response options are mutually exclusive and exhaustive
- [ ] Response scale matches the item content (agreement for opinions, frequency
      for behaviors, intensity for affect)
- [ ] All response points are labeled (not just endpoints)

### Bias
- [ ] No social desirability pull (both high and low responses should feel acceptable)
- [ ] No cultural, gender, age, or ethnic bias in wording or scenarios
- [ ] Negatively keyed items use antonym reversal (not negation)
- [ ] No "absolutes" unless intentional (always, never, completely)

### Discrimination
- [ ] Item should differentiate between high and low scorers on the construct
- [ ] Not so extreme that nearly everyone agrees or disagrees (target: 20%–80%
      endorsement rate for positive items)
- [ ] For cognitive items: difficulty is intentionally placed on the difficulty spectrum

## 2. Scale-Level Quality

Apply to EACH subscale/dimension:

### Content Coverage
- [ ] Items collectively cover the full breadth of the construct domain
- [ ] No critical facet left unrepresented
- [ ] Items span different intensity levels (some easy to endorse, some moderate,
      some extreme)
- [ ] Content expert review confirms comprehensive coverage (CVI targets:
      I-CVI ≥ .78 per item, S-CVI/Ave ≥ .90 for scale)

### Keying Balance
- [ ] Approximately 75% positively keyed, 25% negatively keyed (antonym-based)
- [ ] Reverse items are spread across the instrument (not clustered)
- [ ] OR: alternative bias controls are in place (forced-choice, behavioral language,
      no agree/disagree format)

### Item Count
- [ ] Minimum 5 items per dimension (8–10 preferred)
- [ ] Enough items to achieve target reliability (use Spearman-Brown prophecy formula
      to estimate: rNN = Nr₁₁ / [1 + (N−1)r₁₁])
- [ ] Not so many items that the subscale contains redundant items (watch for
      corrected item-total correlations > .85 between item pairs)

### Predicted Reliability
- [ ] Expected α/ω ≥ .70 for LOW stakes applications
- [ ] Expected α/ω ≥ .80 for MEDIUM stakes applications
- [ ] Expected α/ω ≥ .90 for HIGH stakes individual decisions
- [ ] Estimate using average inter-item correlation: α ≈ Nṝ / [1 + (N−1)ṝ]
      where N = items and ṝ = average inter-item correlation (target ṝ = .15–.50)

### Factor Structure
- [ ] Items are predicted to load on the intended factor (≥ .40 loading)
- [ ] Cross-loadings should be minimal (< .30 on non-target factors)
- [ ] If writing items for an existing validated structure: items match the
      established factor pattern

## 3. Instrument-Level Quality

Apply to the COMPLETE assessment:

### Flow and Respondent Experience
- [ ] Opens with easy, non-threatening items (warm-up)
- [ ] Sensitive items placed in middle third (after rapport, before fatigue)
- [ ] Items grouped by FORMAT (not by construct) to reduce transparent factor structure
- [ ] Within format groups, items are randomized or pseudo-randomized
- [ ] Clear transition between sections (if multiple formats)
- [ ] Estimated completion time is realistic and stated upfront
- [ ] Total length matches stated time: ~2–4 items per minute for Likert,
      ~1–2 items per minute for SJTs/cognitive items

### Validity Architecture
- [ ] Attention check items at 25%, 50%, 75% progress points (if >30 items)
- [ ] Infrequency items included (2–3 extreme implausible statements)
- [ ] Consistency pairs included (2–3 semantically equivalent items placed far apart)
- [ ] Social desirability scale included (for high-stakes applications)

### Administration
- [ ] Standardized instructions are clear and complete
- [ ] Practice items included (for cognitive/SJT formats)
- [ ] Time limits specified (for timed tests) with warning intervals
- [ ] "No Opportunity to Observe" option included (for observer-rated formats)

### Face Validity
- [ ] Assessment looks professional and relevant to the stated purpose
- [ ] Respondents would perceive items as reasonable and related to the topic
- [ ] Assessment title and description match actual content
- [ ] No items that would cause respondents to question the assessment's credibility

### Scoring Completeness
- [ ] Every item has a defined scoring rule
- [ ] All reverse-scored items are clearly marked and the reversal formula is specified
- [ ] Subscale composition is explicitly defined (which items belong to which subscale)
- [ ] Missing data handling rules are specified
- [ ] Score transformation formulas or tables are complete
- [ ] Validity indicator thresholds are defined

## 4. Deliverable Quality

### Assessment Instrument Document
- [ ] Cover page with name, version, date, copyright/license
- [ ] Respondent instructions are clear and complete
- [ ] All items numbered and formatted consistently
- [ ] Response options visually clear and consistently presented
- [ ] Time estimate stated
- [ ] Thank you / completion message at end
- [ ] Accessible design (readable fonts, adequate contrast, screen-reader compatible)

### Administration Manual
- [ ] All 12 sections from the template are present
- [ ] Verbatim administration scripts are provided
- [ ] Scoring procedures are step-by-step and unambiguous
- [ ] Qualified user level is specified
- [ ] Ethical guidelines section is complete
- [ ] Accommodations guidance is included

### Score Interpretation Guide
- [ ] Every dimension has a plain-language description
- [ ] Score ranges map to descriptive labels
- [ ] High and low score behavioral descriptors are provided
- [ ] Common profile patterns are described
- [ ] Limitations and cautions are stated
- [ ] Purpose-specific interpretation guidance is provided

### Individual Report Template
- [ ] All 8 sections from the template are present
- [ ] Visual score display is specified (chart type, colors, labels)
- [ ] Narrative interpretation examples are provided
- [ ] Recommendations are actionable and specific
- [ ] Confidence intervals are included
- [ ] Validity indicators are reported
- [ ] Tone is constructive and strengths-first

### Group Report Template
- [ ] All 6 sections from the template are present
- [ ] Anonymity is protected (minimum n ≥ 5 per subgroup)
- [ ] Statistical summaries are appropriate (means, SDs, distributions)
- [ ] Comparisons use effect sizes, not just significance tests
- [ ] Strategic recommendations are linked to data findings

### Debrief Guide
- [ ] Session structure with time allocations
- [ ] Opening scripts and rapport-building guidance
- [ ] Strengths-first exploration process
- [ ] Development area discussion framework
- [ ] Goal-setting process (SMART format)
- [ ] Cardinal rules listed

## 5. Ethical Compliance

### Informed Consent
- [ ] Consent form template provided
- [ ] Purpose of assessment explained in plain language
- [ ] Voluntary nature stated (for non-selection contexts)
- [ ] Data usage explained (who sees results, how they're used)
- [ ] Data retention period specified
- [ ] Right to withdraw stated (and what happens to data)
- [ ] Right to feedback stated

### Data Privacy
- [ ] GDPR compliance considerations addressed (data minimization, purpose limitation,
      storage limitation, individual rights)
- [ ] Data storage and security recommendations specified
- [ ] Access controls defined (who can see individual results)
- [ ] Anonymization procedures for group reporting specified
- [ ] Data destruction timeline specified

### Fair Use
- [ ] Qualified user level specified (Level A/B/C)
- [ ] Appropriate vs inappropriate uses explicitly listed
- [ ] Non-discrimination statement included
- [ ] Accommodations for special populations documented
- [ ] Warning against use for unintended purposes

### Feedback Rights
- [ ] Right to receive feedback on results is stated
- [ ] Process for providing feedback is defined
- [ ] Qualified interpreter requirement for feedback is specified
- [ ] Dispute/appeal process is defined (for selection contexts)

---

## 6. Final Sign-Off Criteria

An assessment is ready for delivery when ALL of the following are true:

### Minimum Viable Assessment (Must Have)
- [ ] All items pass the item-level quality checklist
- [ ] All subscales meet minimum item count (≥ 5)
- [ ] Predicted reliability meets stakes-appropriate threshold
- [ ] Scoring key is complete and unambiguous
- [ ] Administration instructions are standardized
- [ ] Ethical guidelines are documented
- [ ] Validation recommendations are included (the assessment has NOT been
      empirically validated — this must be clearly stated)

### Professional-Grade Assessment (Should Have)
- [ ] Test blueprint with dimensional weights is documented
- [ ] Validity architecture (attention checks, consistency, infrequency) is in place
- [ ] Score interpretation guide with behavioral descriptors is complete
- [ ] Individual report template with visual profile is designed
- [ ] Administration manual with verbatim scripts is complete
- [ ] Face validity confirmed (items look relevant and professional)

### Excellence Indicators (9+/10)
- [ ] Items span multiple intensity levels across the construct continuum
- [ ] Alternative bias controls beyond reverse scoring are implemented
- [ ] Score reporting includes confidence intervals
- [ ] Cross-cultural considerations are documented (even for single-language versions)
- [ ] Detailed validation roadmap with sample sizes and analyses specified
- [ ] Debrief guide with session structure is provided
- [ ] Group report template with anonymity protections is included
- [ ] Technical appendix template with item analysis framework is prepared
- [ ] Assessment could be submitted to a psychometric reviewer and be taken seriously
